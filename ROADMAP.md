# Roadmap

Pyi-thon is a lightweight browser-first Python learning app. The roadmap keeps the public repo easy to navigate without promising dates that may drift.

## Current Focus

- Keep the app fast, inspectable, and production-ready.
- Improve keyboard access, focus treatment, and reduced-motion behavior.
- Keep release notes, CI, and contributor workflow trustworthy.
- Make sharing and install metadata feel complete.

## Next Candidates

- More beginner-friendly Python levels after level 30.
- Accessibility pass for screen reader labels and modal behavior.
- Progress export and import.
- More language translations.
- More robust server-side AI proxy guidance for shared deployments.

## Label Taxonomy

The repo label set is documented in `.github/labels.yml`.

Use:

- `bug` for broken behavior.
- `enhancement` for product improvements.
- `documentation` for docs-only work.
- `accessibility` for keyboard, focus, screen reader, and contrast work.
- `ui-polish` for visual or motion refinements.
- `curriculum` and `new-level` for lesson content.
- `security` for security-hardening work that does not require private disclosure.
- `dependencies` for dependency and workflow update automation.
- `release` for release workflow, changelog, and tagging work.
- `needs-triage` for unreviewed inbound issues.

## Milestones

Use milestones for release trains, for example:

- `v1.0.0` for the first official release baseline.
- `v1.1.0` for the next small feature release.
- `Backlog` for accepted work that is not assigned to a release.

Keep milestone scopes small enough that a contributor can understand what is expected from the issue list.
