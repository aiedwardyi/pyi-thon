# Support

Pyi-thon is a learning-focused project, so the best support is usually a clear bug report or a well-scoped feature request.

## Before You Open an Issue

- Check the existing issues and pull requests.
- Pick the closest template from the issue chooser.
- Include the smallest reproduction you can.
- Share your browser, OS, and whether you were using dark mode or light mode.

## Good Places to Start

- `Bug Report` for something that breaks or behaves unexpectedly.
- `Feature Request` for an improvement to the learning experience or UI.
- `New Level Proposal` for curriculum ideas or exercise additions.
- `CONTRIBUTING.md` for repo conventions and development guidance.

## What We Can Help With

- Setup or runtime issues
- UI and accessibility problems
- Level design and curriculum questions
- General questions about the repo structure or contribution flow

## What We Cannot Promise

We try to respond when we can, but this is a small public project and support is best-effort. If you need a guaranteed response time, this repo is probably not the right fit.

## Thanks

If Pyi-thon has been useful to you, a star on GitHub is a simple and appreciated way to support the project.
