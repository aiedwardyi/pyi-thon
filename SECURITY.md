# Security Policy

Pyi-thon is a public project, so please avoid sharing secrets, tokens, or exploit details in public issues.

## Supported Versions

We focus on the current `main` branch and the latest tagged release, if one is available.

## Reporting a Vulnerability

Use GitHub Security Advisories if the repository has private vulnerability reporting enabled.

If that option is not available, open a minimal issue that says there is a security concern and request a private follow-up. Do not include:

- API keys
- Full proof-of-concept exploit steps
- Sensitive user data
- Any secret values from your environment

Helpful details to include privately:

- A short description of the issue
- Affected version or commit
- Browser, OS, and environment details
- What impact you observed

## What Happens Next

We will review the report, confirm the issue, and coordinate a fix before sharing details publicly when possible.
